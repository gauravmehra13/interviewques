Q1 What is the use of setTimeout?

  • The global setTimeout() method sets a timer which executes a function or specified piece of code once the timer expires.
  • The time, in milliseconds that the timer should wait before the specified function or code is executed
example:

setTimeout(() => {console.log("Hi, my name is ..")}, 3000);

In this particular example, the output is printed after 3000 ms (3 seconds).

Q2 What is an event loop and call stack?

Event loop:An event loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty.
It follows the queue data structure to maintain the correct sequence in which all operations should be sent for execution.
Event loop gives the illusion of JavaScript being multithreaded, even though it is single threaded, meaning it runs one line at a time.

Call Stack:The call stack is responsible for keeping track of all the operations in line to be executed. Whenever a function is finished, it is popped from the stack. It is like a real stack in data structures where data can be pushed and popped and follows the Last In First Out (LIFO) principle.
Working Of Call stack:

  • When you run a JS fi;e, at the bottommost of the call stack a global execution context is created.
  • Global variables are declared inside GEC and when a function is called, a new execution context is created above GEC
  • Upon function call, the program pauses and control jumps to this context and code is executed line by line.
  • If a declared variable is not present inside the given EC, it searched for it in it's parent EC.
  • After the execution of the code in line by line mannar, the context gets erased and pops out.
  • Then the GEC also gets cleared.

    Q4 What is a Temporal Dead Zone?

    If you are accessing any variable before it's declaration using let and const, you get an error because the variable is in temproal dead zone
    to avoid TDZ , try to access the variable after it's declaration and initialisation.

    Hoisting in JavaScript is a behavior in which a function or a variable can be used before declaration.
    In other words hoisting is JavaScript's default behavior of moving declarations to the top. For example,

    console.log(x)
    var x=8;
