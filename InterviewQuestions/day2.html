Q1 What is creation phase and execution phase?

Creation Phase:When the JavaScript engine executes a script for the first time, it creates the global execution context.
  • During this phase, the JavaScript engine performs the following tasks:
  • Create the global object i.e., window in the web browser or global in Node.js.
  • Create the this object and bind it to the global object.
  • Setup a memory heap for storing variables and function references.
  • Store the function declarations in the memory heap and variables within the global execution context with the initial values as undefined.
  • If a function is called, then the programs pauses in global execution context and control is passed to current execution phase
Execution Phase
  • During the execution phase, the JavaScript engine executes the code line by line, assigns the values to variables, and executes the function calls.
  • For each function call, the JavaScript engine creates a new function execution context.
  • The function execution context is similar to the global execution context. But instead of creating the global object, the JavaScript engine creates the arguments object that is a reference to all the parameters of the function:

Q3 What are function constructors?

    A constructor is a special function that creates and initializes an object instance of a class.
    *In JavaScript, a constructor gets called when an object is created using the new keyword.
    *The purpose of a constructor is to create a new object and set values for any existing object properties.

Example:

function User (name, age) {
    this.name = name;
    this.age = age;
}

let user1 = new User('Gaurav', 25);
let user2 = new User('Bruce Wayne', 27);
console.log(user2)

Q5 What is prototype chain

Every object has a prototype, including the prototype object. This “chain” goes all the way back until it reaches an object that has no prototype, usually Object's prototype. 

    example:
    const electronics= function(){};

    electronics.prototype={
        print(){
            return "These are electronics"
        }
    };

    const mobile= function(){};

    mobile.prototype= Object.create(electronics.prototype);
    mobile.prototype.print= function(){
        return " I am mobile";
    };


    const simCard= function(){};

    simCard.prototype = Object.create(mobile.prototype);
    simCard.prototype.print = function(){
        return "this is a SIM"
    };

    const sanjayElectronics= new electronics();
    const Samsung = new mobile();
    const Airtel = new simCard();

    console.log(sanjayElectronics.print());
    console.log(Samsung.print());
    console.log(Airtel.print());

If a method is not present , it will look up to it's creator in the prototype chain and find the method there.
